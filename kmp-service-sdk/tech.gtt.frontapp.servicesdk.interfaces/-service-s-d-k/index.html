ServiceSDK

expect class ServiceSDK : IServiceSDK

The main entry point for the SDK.

This class provides methods to configure and initialize the SDK. It allows developers to set up necessary configurations, register callbacks, and provide a custom HTTP client for network requests.

Typical usage involves using the Builder class to create and configure an instance of ServiceSDK.

Example Usage

val serviceSDK = ServiceSDK.Builder()
    .setSDKConfiguration(YourSDKConfigurationImpl())
    .setCallback(object : ICallback<SDKResult<Boolean>> {
        override fun call(result: SDKResult<Boolean>) {
            if (result.isSuccess) {
                println("SDK initialized successfully")
            } else {
                println("SDK initialization failed: ${result.error}")
            }
        }
    })
    .build()

With Custom HttpClient

if you want to use your own SDKHttpClient 'YourHttpClient', you can do it like this:

val serviceSDK = ServiceSDK.Builder()
    //set SDKConfiguration and callback
    .setCustomHttpClient(YourHttpClient())
    .build()

Parameters

sdkConfiguration

The configuration information for initializing the SDK. Must include necessary API keys and settings.

callback

A callback for handling the initialization result of the SDK. The callback will provide the result of the initialization process.

httpClient

A custom HTTP client to handle network requests within the SDK. If null, the SDK will use its default HTTP client.
